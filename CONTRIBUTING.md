# Contributing Guidelines

## Criteria for Inclusion

- Actively maintained (commits within the last 12 months)
- Production-ready or has significant community adoption
- Clear documentation
- Not redundant with existing entries

## Entry Format

```markdown
| **Tool Name** | Brief, factual description | [Website](url) Â· [GitHub](url) |
```

## Process

1. Fork and create a branch
2. Add your entry to the appropriate category
3. Submit a PR with a clear title

## Not Accepted

- Abandoned or deprecated tools
- Promotional language
- Duplicate functionality
- Tools without documentation

PRs that don't follow these guidelines will be closed.
